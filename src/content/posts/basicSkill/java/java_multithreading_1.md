# **Java多线程**

## *1.讲一下Java内存模型（JMM）*
- JMM定义了一个主内存，以及每个线程都拥有的工作内存，在对变量进行操作时必须先加载到工作内存中，修改完成后再重新写入主内存中。
- JMM的作用是通过控制变量的加载、写入时机来保障多线程环境下共享变量的安全性。
## *2.在多线程环境下有哪些注意事项*
- （1.）保障线程安全：我们一般从原子性、有序性、可见性三个维度来保障线程安全
    - >原子性：为了保障同一时间只有一个线程访问数据，我们一般通过加锁来实现，java中常见的锁有synchronized关键字和ReentryLock可重入锁。
    - >有序性：CPU为了执行效率，可能会打乱指令预定义的执行顺序，此时我们可以通过添加volatile关键字或者加锁来禁止指令重排序。
    - >可见性: 为了保障当前线程对主内存的修改能够及时被其它线程看见，我们可以通过volatile关键字或加锁的方式强制线程从主内存中读取数据。
- （2.）注意线程的创建与销毁开销：通过线程池使用线程。
## *3.Java中保障线程安全的方式*
- （1.）Synchronized关键字：属于悲观锁，是一种可重入的自动锁，作用是保障被它修饰的方法或者代码块在同一时间只能被一个线程访问。
- （2.）Volatile关键字：属于乐观锁，使用CAS操作实现，作用是保障被它修饰的变量的有序性和可见性。
        
        - 可见性：对于添加了volatile关键字的变量进行操作时，会强制从主内存中读取而不依赖于线程的工作内存。
        - 有序性：对于添加了volatile关键字的变量，会添加内存屏障从而达到禁止指令重排序的效果。
- （3.）Lock接口下的ReentryLock锁：支持手动控制锁的获取与释放时机，可重入，可中断，可以设置重试的超时时间，可设置为公公平锁。

        - ReentryLock的使用方式：
        - 1. 创建对象
        - 2. 加锁（.lock()方法）
        - 3. 释放锁(.unlock()方法)
- （4.）ThreadLocal线程本地变量
    - 原理：每个线程都有一个ThreadLocalMap的成员，它的key用来存放主线程的ThreadLocal对象，value用来存放ThreadLocal的值，这样每个线程就拥有有自己的本地ThreadLocal变量了。
    - 注意事项：

            - ThreadLocalMap可以存放多个主线程的ThreadLocal变量
            - ThreadLocalMap的key是弱引用，可以被自动垃圾回收。ThreadLocalMap的value是强引用，必须被手动释放，否则会造成内存泄漏。
    - 特点：

            - 无法共享，仅当前线程可见
            - 天然安全，用于线程内的数据传递
            - 必须手动释放，否则会造成内存泄漏
- （5.）原子类（AUtomicInteger、AutomicLong）、并发集合（CopyOnWriteArrylist、ConcurrentHashMap）、JUC工具包（Semaphore信号量、CountDownLauch计数器）等
## *4.什么是可重入锁*
- 可重入锁指的是当线程获取到锁之后，在某些情况下需要再次获取该锁时能够直接获取而不产生死锁或其它异常情况。
- 举例来说，当一个对象获取到锁之后，执行方法时又需要重新获取这把锁时能够成功识别并直接获取成功。
## *5.Java中ReentryLock锁的可重入机制*
- ReentryLock内部有一个计数器，当线程获取到锁时计数器会+1，重复获取锁时也不会阻塞，当线程释放锁时计数器-1，当计数器为0时锁才会被释放。
## *6.Synchronized的锁升级机制*
- （1.）无锁状态：当对象刚刚创建，没有任何线程获取时处于无锁状态。
- （2.）偏向锁状态：当一个线程反复获取对象时，会在对象头中设置偏向标记，后续该线程就能够直接获取锁，不需要其他的同步操作。
- （3.）轻量级锁：当两个线程反复交替获取锁时，会在每个线程的栈帧中创建一个锁记录，所记录中会备份一份对象头，然后根据备份的对象头与堆中的对象头做CAS操作，如果成功那么就修改对象头的指向，失败则自旋重试。
- （4.）重量级锁：当有多个线程竞争锁或者轻量级锁的自旋次数超过阈值时会将锁升级为重量级锁，当线程无法获取到锁时会直接阻塞。
## *7.JVM对Synchronized锁的优化*
- （1.）锁膨胀机制：JDK1.6之前Synchronized只有重量级锁这一种状态，JDK1.6添加锁膨胀机制之后，多出了无锁、偏向锁、轻量级锁这三种更常见且性能更好的状态，大幅提高了Synchronized锁的性能。
- （2.）锁消除机制：JVM会对加了锁的代码进行检查，查看代码是否存在多线程竞争的可能性，如果不存在那么就会把锁给消除。
- （3.）锁粗话机制：JVM会对多个连续的加锁、解锁操作进行合并，合并成一个范围更大的锁来便于管理。
## *8.Java中的各种锁*
- （1.）锁的作用：用来解决多线程环境下的线程安全问题，但是加锁会造成阻塞，因此需要平衡好性能与安全问题。
- （2.）锁的类型：

        - 从功能上可以划分为读锁和写锁，也可以说是共享锁和排他锁
            - 读锁（共享锁）：同一时间允许多个线程获取锁
            - 写锁（排他锁、同步锁）：同一时刻只允许一个线程获取到锁
        - 其它锁：
            - 乐观锁：我们认为每次线程访问共享资源时都不会发生冲突，线程可以不停执行，无需加锁也无需等待。
            - 悲观锁：我们认为每次线程访问共享资源时都会发生冲突，因此每次进行操作时都需要加锁，以保障同一时间只有一个线程访问共享资源。
            - 偏向锁/轻量级锁/偏向锁
            - 公平锁/非公平锁：公平锁指的是线程按照申请锁的顺序获取锁，非公平锁则是存在线程插队抢占锁，非公平锁整体执行效率更高但是可能会导致线程饥饿。
            - 可重入锁
            - 分布式锁：解决分布式架构下的线程安全问题。
