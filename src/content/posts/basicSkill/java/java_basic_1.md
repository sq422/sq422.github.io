# **Java基础**

## *1.java的特点*
- （1.）平台无关性：java编译器会将源程序编译成.class字节码文件，该字节码文件可以在任何安装了JVM的平台上运行。
- （2.）面向对象：java中一切皆对象，相关的特点有：封装、继承、多态、类、对象等等。
- （3.）自动内存管理：java有自己的内存管理机制，能够自动对不再被引用的对象进行垃圾回收以及内存管理。
## *2.java的优点和缺点*
-   优点：平台无关、面向对象、自动内存管理、支持多线程编程、拥有丰富且强大的生态。
-   缺点：性能较差、语法复杂、内存消耗过大、面向对象过于复杂。
## *3.JVM的作用*
- （我们编写的java源代码在被编译之后会生成.class字节码文件）JVM的作用就是将字节码文件翻译成平台对应的机器码，并执行程序。JVM还支持内存管理、垃圾回收等功能。
## *4.JVM、JRE、JDK之间的区别*
- （1.）JVM：jvm是java虚拟机，它的作用是将字节码文件翻译成平台对应的机器码并运行，同时还支持内存管理和垃圾回收等功能。
- （2.）JRE：jre是java运行时环境，它是提供了java运行所需的最小环境，它只包含JVM和一组核心类库，不包含java开发和调试等一系列其他工具。
- （3.）JDK：jdk指的是java开发工具包，它提供了java程序开发、编译、调试、运行等一系列操作所需的所有工具与环境。
## *5.怎么理解面向对象*
-   面向对象：面向对象是一种编程思想，它将一切事物都看作对象，对象拥有自己的属性和行为方法。
-   面向对象编程：面向对象编程以对象为中心，通过对象之间的行为交互来实现程序的功能，具有很好的灵活性与可拓展性。
## *6.封装、继承、多态*
-   （1.）封装：封装指的是将对象的属性和行为方法结合在一起，对外部隐藏具体的实现细节，仅提供接口与外部进行交互。封装的目的是增强程序的安全性和简化开发，使对象更加独立。
    - >安全性：对于属性来说，通过访问控制符可以保障核心数据不被外部篡改。对于方法来说，通过封装隐藏了具体的实现细节也能够保障逻辑不被外部程序修改。
    - >简化开发：通过封装将复杂的内部实现逻辑封装成一个黑盒，开发者不需要关系内部是如何实现的，只需要了解对象暴露出来的接口有什么作用。
-   （2.）继承：继承是一种使子类能够复用父类的数据结构和方法的机制。继承的目的是增强代码的可复用性，并建立清晰的层次结构。
-   （3.）多态：多态指的是将相同的行为方法赋予不同的对象会产生不同的结果。多态的作用是增强程序的灵活性和可拓展性。在java中多态又分为运行时多态和编译时多态。
    - >灵活性：父类的引用可以接收不同的子类实例，调用同样的方法后表现出的行为是不同的，通过这种方式我们可以通过相同引用指向不同实例来灵活地调用各种逻辑。
    - >可拓展性：通过多态我们可以实现在不修改原有代码的基础上实现功能增强，具体来说就是我们原有代码中的一个引用指向了子类实例，那么我们就可以通过修改子类方法来对原有代码实现功能增强。
## *7.多态性体现在那几个方面*
- （1.）运行时多态：方法重写，在子类中重写父类的方法。
- （2.）编译时多态：方法重载，在同一个类中编写方法名相同但是参数列表不同的方法。
## *8.基于多态性的向上转型和向下转型*
- （1.）向上转型：向上转型是安全的，用于隐藏子类特有的属性和方法，仅暴露父类定义的属性和方法。
- （2.）向下转型：向下转型是不安全的，需要显式声明强制进行类型转换，一般用于恢复对子类特有方法的访问能力。
## *9.java中的访问控制权限*
1. private：仅当前类可见
2. default：当前包中的类可见
3. protected：当前包与当前类的子类可见
4. public：所有类可见
## *10.普通类、抽象类、接口的区别*

| 特性 | 普通类 | 抽象类 | 接口 |
|:------|:-----:|------:|------:|
| 是否可以实例化 | 可以 | 不可以 | 不可以 |
| 继承/实现方式 | extends关键字单继承 | extends关键字单继承 | implements关键字多实现 |
| 有无构造方法 | 有 | 有 | 无 |
| 方法实现 | 必须实现 | 可以有普通方法也可以有抽象方法 | 一般只有抽象方法（java8之后可以有其他方法） |
| 成员变量 | 各种类型 | 各种类型 | 只能是常量 |

## *11.Java中的关键字*
1. final关键字：用于禁止类被继承、方法被重写、变量被修改。
2. static关键字：
    - 静态变量：被static关键字修饰的变量被称为静态变量，它又被称为类变量，因为它属于整个类而不是某个具体的对象实例。
    - 静态变量的特点：

            - 所有实例共享静态变量
            - 静态变量在类加载时被初始化而非在实例化时被初始化
            - 静态变量可以通过类名进行访问
    - 静态方法：被static关键字修饰的方法
    - 静态方法的特点：

            - 无实例依赖，不需要实例化即可通过类名进行使用
            - 只能访问静态成员（静态变量或者方法）
## *12.Java中的基本数据类型*
1. 数值型：byte(1),short(2),int(4),long(8),float(4),double(8)
2. 字符型：char(2)
3. 布尔型：bool(1)
## *13.Java中的数据转换类型*
1. （1.）自动类型转换：当目标数据类型的范围大于原数据类型时，java会进行自动类型转换。
2. （2.）强制类型转换：当目标数据类型的范围小于原数据类型时，需要显示声明进行强制类型转换。
3. （3.）基本类型-包装类型转换：java中会自动进行装箱（Integer.valueOf(basicInt)）与拆箱（wrapperNum.intValue()），也可手动进行。
4. （4.）字符型-数值型转换：java中提供了能够将字符型数据转换成数值型数据的各种方法，比如说Integer.parseInt().
## *14.Java为什么要使用包装类型*
1. （1.）包装类型将基本数据与处理数据的方法结合在了一起，能够方便使用
2. （2.）在java中绝大部分类和工具都是用来处理引用数据类型的，而无法处理基本数据类型

    - >ArrayList只能存储引用数据类型，而无法存储基本数据类型的对象
    - >泛型也只能使用引用数据类型，无法使用基本数据类型
## *15.Integer和int的区别*
1. （1.）int是预定义的基本数据类型，不需要实例化，而Integer是包装类，需要实例化
2. （2.）int不需要额外分配内存，Integer需要分配内存实例化
3. （3.）基本数据类型的性能比包装类型更好
## *16.Integer缓存*
- Java会对-128到127的整形对象进行缓存，当时有Integer.valueOf()方法获取整形对象时，会优先从缓存池中获取，如果不存在才会创建新的对象。
## *17.为什么使用BigDecimal而不使用double*
- 因为double执行的是二进制浮点数运算，当遇到二进制无法表示的数时可能会出现精度丢失的问题。（比如说二进制就无法表示0.1）
- 而BigDecimal是专门为高精度十进制运算设计的类，不会出现精度丢失的问题。
