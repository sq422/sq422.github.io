# **Mysql基础知识**

1. 1.*对Mysql的理解以及它的特点*：
    - Mysql是一款基于磁盘的关系型数据库，它支持事务与隔离级别，具有跨平台性与兼容性。
    - 优点：
        - （1.）基于磁盘的存储使它支持海量数据的存储
        - （2.）它存储数据的数据结构（表结构）使它能够对数据进行高效地管理（增删改查）
        - （3.）InnoDB实现的MVCC和锁机制保障了事务与隔离级别的实现，使它能够保障高并发线程安全
        - （4.）开源免费、成本极低
        - （5.）社区完善、生态丰富
    - 缺点：
        - （1.）使用Mysql需要对磁盘进行IO操作，读写速度较慢
        - （2.）数据结构比较复杂，对数据的操作较慢
        - （3.）单表性能上限，当单表数据达到千万级时，即使有索引，读写性能也会显著降低
2. 2.*创建索引时有什么注意事项*:
    - （1.）控制索引数量：索引并不是越多越好，因为维护索引会产生性能开销，导致系统性能变差。
    - （2.）优先为高频查询字段添加索引：因为索引最强大的功能就是提高查询效率。
    - （3.）避免为低频查询或高频更新的字段添加索引：因为频繁地更新操作会导致索引频繁地被维护，性能开销较大。
    - （4.）联合索引优先：如果需要查询到数据能够直接在索引中被找到，就可以避免回表查询，大大提高了查询效率。
3. 3.*什么是回表*：
    - 回表操作指的是非覆盖索引无法直接从索引中获取所需要的值，因此会通过查询到的主键值到主键索引中去查询行数据信息。
    - > 在InnoDB引擎中，数据表的主键会默认创建聚簇索引，这个索引的非叶子结点仅存储索引值，叶子结点才会存储具体的行数据。
    - > 非聚簇索引又被称为二级索引，二级索引的叶子节点存储的数据是主键值，可以根据这个主键值进行回表操作。
4. 4.*前缀索引*：
    - 前缀索引是一种针对于字符数据的优化手段，它不索引全量字符，而是仅索引字符数据的前面一部分数据。（char、text）
    - > 使用前缀索引可以节省内存空间、提高写入性能，同时在大多数情况下也能保证查询性能。
